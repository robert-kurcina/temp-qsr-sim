# MEST Tactics: Core Data Concepts

This document outlines the fundamental data structures that form the basis of the MEST Tactics simulation engine.

## 1. Archetype

An **Archetype** is the foundational template for any unit in the game. It represents a specific type of combatant, defined by its inherent capabilities.

- **Definition:** An Archetype consists of:
    - A set of base **Attributes** (CCA, RCA, REF, etc.).
    - A list of inherent **Traits**.
    - A base **Build Point (BP)** cost.
    - A **Species** (e.g., "Humanoid").
    - An optional **Class** (e.g., "Common", "Variant Common").

- **Source:** Archetype data is loaded directly from `src/data/archetypes.json`.

- **Analogy:** Think of an Archetype as the "class" or "job" of a character in a role-playing game (e.g., "Warrior", "Mage").

## 2. Profile

A **Profile** is an Archetype that has been equipped with items. It represents a complete character loadout, ready to be deployed.

- **Definition:** A Profile is composed of:
    - A base **Archetype**.
    - A list of **Items** (weapons, armor, equipment).
    - **Calculated Statistics:** The Profile includes numerous calculated values that are derived from the combination of the Archetype and its Items. These include:
        - `totalBp`: The total Build Point cost of the Archetype and all its Items.
        - `adjustedBp`: The BP cost after applying discounts for multiple weapons.
        - `physicality`, `durability`, `burden`, `totalHands`, `totalDeflect`, `totalAR`: Combat-related stats derived from attributes and items.
        - `finalTraits`, `allTraits`: The complete list of traits from both the Archetype and its Items.

- **Source:** Profiles are generated by the `profile-generator.ts` script, which combines an Archetype with a set of Items.

- **Analogy:** A Profile is like a completed character sheet, with all the stats and equipment listed, ready for play.

## 3. Character

A **Character** is a unique, individual instance of a Profile that exists on the battlefield. 

- **Definition:** A Character is defined by:
    - A unique **`id`**.
    - A **`name`**.
    - A **`profile`**, which provides all of its base stats and equipment.
    - A **`state`** object, which tracks its current condition in the game (wounds, delay tokens, fear tokens, status effects, etc.).

- **Source:** Characters are created during the game simulation. They are instantiated using a Profile.

- **Analogy:** A Character is the actual game piece on the board, representing a specific individual with their own name and ongoing state.

## Summary of the Hierarchy

**Archetype** (the template) -> **Profile** (the character sheet) -> **Character** (the individual on the battlefield)
