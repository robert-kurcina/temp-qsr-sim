# Advanced Rules: Fire (▲)

**Status:** ⏳ **DEFERRED** - Requires core QSR implementation plus Fire trait support in `traits_descriptions.json`.

---

## Overview

Fire is a **dynamic terrain feature** generated by certain traits, spells, weapons, or Items of equipment. Fire markers represent burning areas that cause damage and obscure vision.

---

## Fire Markers

### Marker Sizes

| Fire Points | Diameter | Height | Light X |
|-------------|----------|--------|---------|
| **1** | 0.75 MU | ~1.5 MU | 7 |
| **2** | 1 MU | ~2 MU | 8 |
| **3** | 2 MU | ~4 MU | 9 |
| **4** | 3 MU | ~6 MU | 10 |

**Notes:**
- Fire markers generate **Obscuring terrain** within 1" up to **twice its base-diameter** in height
- Fire markers behave as **Soft Cover**
- Models may **cross** but **not stand upon** Fire markers

### Light Generation

Fire markers generate **Light X** where **X = Fire points + 6**.

**Additional Light** is generated in any given **4 MU diameter area**:
- **+1** for a second marker of equal X
- **+1** for four markers of X minus 1

## Fire Effects

### Scorch Test

At the **start of a character's Initiative**, determine how many Fire markers are in **base-contact**.

**Test Sequence:**
1. **Scorch Test** — Unopposed REF Test at **-1m per Fire marker** in base-contact
2. **Damage Test** — Upon fail, Unopposed SIZ Test as Damage Test, adjusted by **-1m per miss**
3. **Burned Tokens** — Assign **1 Burned token per miss** on Damage Test

```
Example:
Character in base-contact with 2 Fire markers
Scorch Test: REF vs System, -2m penalty
If fails by 3 misses:
  Damage Test: SIZ vs System, -3m penalty
  If fails by 2: assign 2 Burned tokens
```

## Spreading Fire

Fire markers in **base-contact with Flammable terrain** may **"Spread Fire"**.

### Spread Check

Check at the **end of each Turn** for every Fire marker:

**Dice Pool:** 2 Modifier dice
- **+1 Modifier die** if Fire marker is already **size 4 (3 MU)** diameter
- **+1 Modifier die** if spreading **upwards**

**Results:**
| Successes | Effect |
|-----------|--------|
| **1+** | Increase Fire marker size by 1 |
| **2+** | Place size 1 Fire marker in base-contact at Flammable terrain |
| **3** | Also generate **Gas:Blacksmoke Puff** marker in base-contact |

**Placement:** Use **Unbiased Scatter diagram** if necessary for new markers.

## Reducing Fire Markers

Fires disappear by being **"reduced"** at the **end of the current Player's Turn**.

### Automatic Reduction

| Terrain Type | Reduction |
|--------------|-----------|
| **Indoors** (linoleum, marble, concrete, cement, sand, rocks, dirt) | -1 size before rolling |
| **Wet terrain** (water, rivers, ponds, snow) | -2 sizes before rolling |

### Reduction Roll

**Dice Pool:** 2 Modifier dice

**Results:**
| Successes | Effect |
|-----------|--------|
| **0** | Reduce size by 1 |
| **1+** | No reduction |

**Removal:** Remove Fire marker if reduced below **size 1**.

### Wet Terrain Interaction

- Fire markers **never cross** into Wet terrain
- Fire markers on Wet terrain replaced with **Gas:Steam markers** of equal size

## Gas and Fire Interaction

### Gas Movement

**Gas markers never move towards Fire markers.**

When a Fire marker is placed:
1. All Gas markers **within their own base-diameter** of the Fire marker **reposition away** by the Fire marker's base-diameter
2. Start with **largest Gas markers first**, then next size smaller
3. If Attacker and target location known, reposition **away from LOF**
4. Otherwise use **Unbiased Scatter diagram**

### Gas Stacking

Gas markers that become **in base-contact** with other Gas markers:
- Reposition those others in the **same direction**
- **Remain in base-contact**
