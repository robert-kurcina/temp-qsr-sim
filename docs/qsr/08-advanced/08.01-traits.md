## Runtime-Imperative Open Gaps (Post Classification)

1. Advanced-effects Hindrance penalties are not yet fully integrated into the core hindrance penalty path (`subroutines/hindrances.ts` currently keys only Wound/Fear/Delay).
2. ROF/Suppression helper logic is now partially connected to the authoritative combat action pipeline (`executeRangedAttack` + `executeSuppressAction` in `GameManager`), but full clause-accurate lifecycle semantics (including all flip/retention conditions and broad action-path coverage) are still incomplete.
3. `calculateROFMarkerPositions(...)` no longer hardcodes ally detection to `false`, but still relies on side-token metadata heuristics (`sideId`/`teamId`/`factionId`) rather than a canonical side-registry callback from the turn engine.
4. `UtilityScorer` ROF/Suppression safety scoring still initializes marker arrays as empty placeholders, so suppression-zone evaluation is not yet fed by live battlefield marker state.
5. Firelane role/status clauses (Braced/Emplaced lifecycle, Gunner/Loader constraints, Team-work transfer) are not yet owned by a dedicated runtime action/status subsystem.
6. Advanced fire rules (spread/reduce/gas coupling/terrain conversion) currently have guide parity but no authoritative marker lifecycle subsystem in runtime.
7. Advanced lighting rules (internal lighting controls, point-light attenuation, and light-casting geometry) are not yet represented by a runtime lighting subsystem beyond simplified global visibility presets.
8. Advanced gas/fume/puffs rules currently have guide parity but no runtime gas-marker lifecycle subsystem (spread/grow/reduce/linger/reposition).
9. Advanced webbing rules currently have guide parity but no runtime strand-marker subsystem (placement, morass penalties, removal/decay, fire interaction).
10. Advanced-technology imperative scope is still narrow (`ATE.3` partial): runtime enforcement currently focuses on tech-window item filtering and does not yet provide configurable cross-age exception policy controls.
11. Logic-only advanced-trait test surface was reduced (`advanced-traits.test.ts` now runtime-backed), but multiple advanced domains still lack full engine-path enforcement tests.
12. Go Points economy and Group Action lifecycle are only helper-level; there is no turn-sequence GP acquisition/spend ledger and `GameManager` group attack entry points are still TODO stubs.
13. Champions/LoA campaign semantics (reputation progression, intent test, escape/CVP, champion-based LoA exceptions) are not yet runtime-owned.
14. Buildings access-state semantics (door/window/lock lifecycle, lockpicking, stair/ladder traversal markers) are not yet runtime-owned.
15. Advanced terrain Tricky-keyword semantics (`Uneven`, `Mire`, `Wet`, `Water`, `Flammable`) are not yet owned by a runtime terrain-keyword subsystem.
16. All `Advanced-*.txt` canonical files are now clause-mapped as at least `Partially Audited`; remaining work is closure of the `72` runtime-imperative clauses.
17. The `9` narrative/setup clauses are intentionally docs-level by this classification and are not part of runtime-closure targets.
